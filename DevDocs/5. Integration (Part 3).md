# Integration Guide (Part 3): Power & Security

In the final part of this guide, we cover how to leverage Pezhvak's adaptive power profiles and how to implement the "Panic Button" for immediate data destruction.

## 1. Managing Adaptive Power Levels

Pezhvak Core allows the UI to dynamically scale the radio's performance. As a GUI developer, you should expose these levels to the user based on their current needs (e.g., a "Turbo" toggle or a battery-saving mode).

### Setting the Power Level
Call `SetRadioPowerLevel(level)` to update the engine's fragmentation logic and signal the native radio.

*   **0 (Normal):** Default. Low battery impact. Smaller packets (200 bytes).
*   **1 (High):** Balanced. Better range/speed. Larger packets (450 bytes).
*   **2 (Max):** Extreme. Full throttle. Maximum MTU (480 bytes) and no inter-packet delay.

```kotlin
// Android Example: Switching to Max Power
try {
    pezhvak.setRadioPowerLevel(2) 
    // This also triggers MyBluetoothPlatform.setRadioPowerLevel(2)
} catch (e: Exception) {
    // Handle cases where the OS might reject high-power BLE requests
}
```

### Reflecting State in the UI
You can query the current state to update icons or status bars:
```swift
// iOS Example
if pezhvak.isHighPowerEnabled() {
    self.statusIcon.image = UIImage(named: "bolt.fill")
}

let currentMtu = pezhvak.getCurrentPayloadSize()
print("Current throughput: \(currentMtu) bytes/packet")
```

## 2. Implementing the "Panic Button"

The `WipeAllData()` method is a critical security feature. It performs an immediate, destructive clear of the local database.

### What happens during a Wipe?
1.  All message reassembly buffers in the Router are cleared.
2.  All pending relay messages in BadgerDB are deleted.
3.  All "seen" deduplication logs are removed.

### Important: Identity Cleanup
While `WipeAllData()` clears the mesh database, the Core does **not** delete your `identity.json` file automatically (as it doesn't own your filesystem). Your UI must handle this:

```kotlin
fun triggerPanicButton() {
    // 1. Clear the mesh engine
    pezhvak.wipeAllData()
    
    // 2. Delete the identity file
    val identityFile = File(context.filesDir, "identity.json")
    if (identityFile.exists()) {
        identityFile.delete()
    }
    
    // 3. Force exit or restart to a "Setup" screen
    System.exit(0)
}
```

## 3. Security Best Practices

*   **Key Storage:** Never store the `private_key` in plain text outside of the app's internal private storage. On Android, use the EncryptedSharedPreferences; on iOS, use the Keychain.
*   **Background Execution:** BLE sync works best when the app is in the foreground. If your app goes to the background, the OS may throttle the radio. Ensure your `NativePlatform` implementation handles "Connection Refused" errors gracefully by letting the Core save messages for the next foreground sync.
*   **Screen Security:** For high-security users, consider implementing "Screen Shielding" in your UI to prevent sensitive messages from appearing in system screenshots or task switchers.
