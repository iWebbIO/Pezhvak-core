# Integration Guide (Part 1): Foundation

Integrating Pezhvak Core into your mobile or desktop application involves three main components: initializing the database, implementing the hardware bridge (NativePlatform), and starting the Core engine.

## 1. Setting up the Storage

Pezhvak Core uses **BadgerDB** for message persistence. Before initializing the core, you must provide a writable directory path where the database can store its files.

**Note for Mobile:** Ensure you use the internal "Files" directory of your app (e.g., `context.getFilesDir()` on Android or `FileManager.default.urls(for: .documentDirectory, in: .userDomainMask)` on iOS) to ensure data is sandboxed and protected.

```go
// Example of initializing the store in Go (or via gomobile)
db, err := core.NewBadgerStore(dataDirPath)
if err != nil {
    // Handle database initialization failure
}
```

## 2. Implementing the NativePlatform Interface

The `NativePlatform` interface is the "contract" between the Go engine and your application's Bluetooth stack. You must implement this in your native language (Kotlin/Swift).

### The Interface Definition
```go
type NativePlatform interface {
    SendBLE(peerID string, data []byte) error
    SetRadioPowerLevel(level int) error
    OnMessageReceived(senderID string, plaintext []byte)
}
```


## 3. Initializing the Core

Once you have your `MessageStore` and your `NativePlatform` implementation, you can instantiate the Core. You will need the user's hex-encoded public and private keys (usually loaded from an `identity.json` file).

```kotlin
// Android example
val pezhvak = core.Core.newPezhvakCore(
    myPlatform,     // Your NativePlatform implementation
    myBadgerStore,  // The initialized BadgerStore
    privateKeyHex, 
    publicKeyHex
)
```

## Critical Lifecycle Management

Because the Core manages background threads and database locks, you **must** call `.close()` when the application is being terminated or the service is stopped.

```kotlin
override fun onDestroy() {
    pezhvak.close()
    super.onDestroy()
}
```
